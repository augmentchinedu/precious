# AI Project Documentation Policy

## 1. Purpose

This document outlines the structure, standards, and conventions for all documentation related to the `ai` project. Adherence to this policy ensures our knowledge base remains organized, accessible, and maintainable. This policy is managed by the Assistant, Beauty.

## 2. Directory Structure

All documentation for this project MUST reside within `docs/projects/ai/`. The standard structure is as follows:

```
docs/projects/ai/
├── ARCHITECTURE.md
├── DECISION_LOG.md
├── DOCUMENTATION_POLICY.md
├── ONBOARDING.md
├── SPECIFICATIONS/
│   └── *.md
└── MEETINGS/
    └── YYYY-MM-DD_summary.md
```

## 3. Core Documents

*   **ARCHITECTURE.md**: The single source of truth for the high-level system design, architectural principles, and component interactions. Managed by the Project Architect, Michael.
*   **DECISION_LOG.md**: An immutable log of significant architectural decisions, clarifications, and answered questions. This provides historical context for the project's evolution.
*   **DOCUMENTATION_POLICY.md**: This file. It defines the rules for project documentation.
*   **ONBOARDING.md**: A guide for new developers or agents joining the project, explaining how to get started.

## 4. Subdirectories

*   **/SPECIFICATIONS**: Contains detailed design documents for specific features, components, or agent capabilities. Each specification should be in its own Markdown file.
*   **/MEETINGS**: Contains summaries and key outcomes of project-related meetings.

## 5. File Naming and Formatting

*   File names should be `UPPERCASE_SNAKE_CASE` for primary documents and `kebab-case` or `DescriptiveName` for specifications.
*   All documents must use standard Markdown.
*   All documents must start with a level-one heading (`# Title`).


*   **DEVELOPER_GUIDELINES.md**: A set of operational procedures and best practices for developer agents. It covers the development cycle, testing, and file action protocols. Managed by the Developer Operator, Roni.


*   **WORK_ITEMS.md**: A list of discrete tasks and work items for the `ai` project. It serves as a simple project backlog. Managed by the Developer Operator, Roni.


*   **COMMUNICATION_STRATEGY.md**: Outlines the narrative, key themes, and strategy for communicating the `ai` project's progress to an external audience. Managed by the Social Media Manager, Francesca.
